# Department Management API - Documentation

یہ ڈاکومنٹیشن ڈیپارٹمنٹ (Department) ماڈیول کے کام کرنے کا طریقہ کار واضح کرتی ہے۔ اس میں ہم نے پروجیکٹ کی دوسری بڑی ریکوائرمنٹ (Feature 2) کے مطابق مکمل CRUD آپریشنز، صفحہ بندی (Pagination) اور سرچنگ (Searching) شامل کی ہے۔

## 🚀 فیچرز اور ان کا کام (Core Features)

### 1. نیا ڈیپارٹمنٹ بنانا (Create Department)
* **Endpoint:** `POST /api/v1/departments`
* **کام کا طریقہ:** نیا ڈیپارٹمنٹ بناتے وقت کلائنٹ صرف نام (Name) بھیجتا ہے۔ سسٹم خود بخود اس کے لیے ایک محفوظ `UUID` جنریٹ کرتا ہے اور اسے ڈیٹا بیس میں محفوظ کر لیتا ہے۔

### 2. تمام ڈیپارٹمنٹس کی لسٹ دیکھنا (Get All Departments with Pagination)
* **Endpoint:** `GET /api/v1/departments`
* **کام کا طریقہ:** یہ API پروجیکٹ کی ریکوائرمنٹ کے عین مطابق تمام ڈیپارٹمنٹس کو ایک ساتھ لانے کے بجائے **Pages (صفحات)** میں لاتی ہے۔ اس سے سرور کی پرفارمنس بہترین رہتی ہے۔

### 3. کسی ایک ڈیپارٹمنٹ کا ریکارڈ دیکھنا (Get Single Department)
* **Endpoint:** `GET /api/v1/departments/{uuid}`
* **کام کا طریقہ:** مخصوص `UUID` کے ذریعے کسی بھی ڈیپارٹمنٹ کا ڈیٹا سیکنڈوں میں تلاش کر کے لاتی ہے۔

### 4. ڈیپارٹمنٹ کا نام تبدیل کرنا (Update Department)
* **Endpoint:** `PATCH /api/v1/departments/{uuid}`
* **کام کا طریقہ:** اگر کسی ڈیپارٹمنٹ کا نام تبدیل کرنا ہو، تو اس API کے ذریعے پرانا نام نئے نام سے اپڈیٹ کیا جا سکتا ہے۔

### 5. ڈیپارٹمنٹ کو ڈیلیٹ کرنا (Soft Delete)
* **Endpoint:** `DELETE /api/v1/departments/{uuid}`
* **کام کا طریقہ:** ہم ڈیپارٹمنٹ کو ہارڈ ڈیلیٹ (ہمیشہ کے لیے ختم) نہیں کرتے کیونکہ اس ڈیپارٹمنٹ میں کئی ملازمین کام کر رہے ہو سکتے ہیں۔ ہم صرف اس پر `isDeleted = true` کا فلیگ لگا دیتے ہیں تاکہ پرانا ریکارڈ خراب نہ ہو۔

### 6. ڈیپارٹمنٹ کو نام سے تلاش کرنا (Search by Name)
* **Endpoint:** `GET /api/v1/departments/search?name={نام}`
* **کام کا طریقہ:** یہ API نام کے کسی بھی حصے (Partial Match) سے ڈیپارٹمنٹ تلاش کر سکتی ہے اور Case-Insensitive (چھوٹے بڑے حروف کی تمیز کے بغیر) کام کرتی ہے۔ اس کا رزلٹ بھی Pagination کے ساتھ آتا ہے۔

---
## 🛡️ سیکیورٹی (Security Logic)
ایمپلائی ماڈیول کی طرح، یہاں بھی ہم نے ڈیٹا کے تبادلے کے لیے **DTOs** اور سیکیورٹی کے لیے **UUIDs** کا استعمال کیا ہے تاکہ ڈیٹا بیس کے اصلی آئی ڈیز (IDs) کلائنٹ سے چھپے رہیں۔
